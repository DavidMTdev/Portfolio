Article N°4

15/07/2019

Création d’un programme

Ce jour-là, ma tutrice m’a confié une mission qui consistait à automatiser une tâche manuelle. Dans un premier temps j’ai étudié le problème que ma tutrice m’a octroyé et trouvé rapidement plusieurs solutions à ce problème, dont j’ai choisi celle où mes compétences pour la réaliser était les plus élevé.

Afin de réaliser cette mission j’ai dû apprendre et installer des modules et des librairies du langage python. Lors de l’apprentissage de ces modules j’utilisais la version 3.7 de python sous une machine 64 bits. De plus j’ai utilisé un module pour interagir avec la base de données oracle et un autre module qui permet de compiler et créer un exécutable du programme. Après avoir appris et installé les modules j’ai pu commencer la réalisation du script.

Le jour suivant, le script fut terminé et j’ai pu commencer des tests sur le serveur, mais j’ai rencontré quelques erreurs sur le serveur, or sur ma machine le programme fonctionnait. De longues heures de recherches ont été nécessaire pour comprendre l’erreur qui me posait problème, dont la solution était que j’utilisé python 3.7 64 bits sous une machine 32 bits. Pour faire face à cette erreur j’ai dû recompiler mon script avec python 3.7 32 bits. Pour donner suite à cela j’ai effectué à nouveau un test sur le serveur mais une nouvelle erreur est apparue. Cela m’a demandé de nouvelles heures de recherche pour résoudre le nouvel problème qui se posait. J’ai alors compris que l’OS du serveur était un Windows serveur 2003 32 bits, or j’utilisais python 3.7. Toutefois, l’OS du serveur est trop ancien pour l’utilisation de python 3.5 ou plus, c’est pour cela que je me suis mis à utiliser python 2.7 32 bits. Par ailleurs, j’ai remarqué que la base de données utilisé Oracle 9.2, ce qui a nécessité que j’utilise un autre module pour interagir avec la base de données, car le module utilisé auparavant ne fonctionnait pas avec le module 9.2. Enfin, les nouveaux testes effectués ont été concluant, la mission était résolue.
